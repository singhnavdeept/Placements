## SUPPORT VECTOR MACHINES (SVM)

**Definition:** A powerful supervised learning algorithm used for classification and regression. It works by finding the optimal hyperplane (decision boundary) that best separates data points of different classes in an n-dimensional space. The goal is to maximize the **margin**—the distance between the hyperplane and the nearest data points from each class.

**Used For:**

- Classification (SVC - Support Vector Classification)
    
- Regression (SVR - Support Vector Regression)
    
- Outlier detection (One-Class SVM)
    
- Ranking problems
    
- Complex non-linear data classification (using kernels)
    

**Cases of Application:**

- Face detection (classifying parts of image as face/non-face)
    
- Text and hypertext categorization
    
- Bioinformatics (protein classification, cancer classification)
    
- Handwriting recognition
    
- Generalized predictive control (GPC)
    
- Stock market forecasting
    
- Image classification
    

**Prerequisites:**

- **Feature Scaling is mandatory** (SVM is distance-based)
    
- Numerical inputs (categorical data must be encoded)
    
- No missing values (must be handled beforehand)
    
- Typically binary classification (extended to multiclass via strategies)
    
- Smaller to medium-sized datasets preferred (due to training time)
    

**Performance Measures:**

- **Classification:** Accuracy, Precision, Recall, F1-Score, ROC-AUC, Hinge Loss
    
- **Regression:** Epsilon-Insensitive Loss, MSE, R² Score
    
- **Model-specific:** Number of Support Vectors, Margin width

## HOW SVM WORKS

**Algorithm Overview:**

**Training Phase (Finding the Hyperplane):**

1. Plot data points in n-dimensional space (where n is number of features).
    
2. Identify **Support Vectors** (the data points closest to the potential separating line).
    
3. Find the hyperplane that maximizes the distance (margin) between these Support Vectors.
    
4. If data is not linearly separable, project it into a higher-dimensional space using the **Kernel Trick** to make it separable.
    
5. Optimize the cost function (Hinge Loss + Regularization) to balance margin width and misclassification errors.
    

**Prediction Phase:**

1. Project the new data point into the same feature space.
    
2. Calculate which side of the hyperplane the point falls on.
    
3. Assign class label based on the side ($+1$ or $-1$).
    

**Key Concept:** "The widest street approach." Imagine trying to draw a road (margin) between two groups of houses (classes) such that the road is as wide as possible without hitting any house.

## CORE COMPONENTS

### **1. Hyperplane**

- The decision boundary that separates classes.
    
- In 2D: A line.
    
- In 3D: A flat plane.
    
- In >3D: A hyperplane.
    
- Equation: $w \cdot x + b = 0$
    

### **2. Support Vectors**

- The specific data points closest to the hyperplane.
    
- They "support" or define the orientation and position of the hyperplane.
    
- - **Crucial:** If you remove other points, the model doesn't change. If you remove a support vector, the hyperplane shifts.
        

### **3. Margin**

- The distance between the hyperplane and the nearest data points (support vectors).
    
- **Goal:** Maximize this margin.
    
- **Hard Margin:** Strictly no misclassifications allowed (works only for perfectly separable data).
    
- **Soft Margin:** Allows some misclassifications to handle noise and non-linear data (controlled by hyperparameter `C`).
    

---

## MATHEMATICAL INTUITION (THE "SPLITTING" CRITERIA)

Unlike Decision Trees which use entropy, SVM uses geometric optimization.

### **1. The Objective Function**

We want to maximize the margin ($\frac{2}{||w||}$) while keeping classification correct. This is mathematically equivalent to minimizing the norm of the weight vector $w$.

Formula:

$$\min_{w,b} \frac{1}{2} ||w||^2$$

Subject to constraints:

$$y_i (w \cdot x_i + b) \ge 1$$

(For every data point $i$, the class label $y_i$ multiplied by the position must be on the correct side of the margin)

### **2. Hinge Loss (The Cost Function)**

Used when data isn't perfectly separable (Soft Margin). It penalizes points only if they are on the wrong side of the margin.

Formula:

$$Loss = \max(0, 1 - y_i(w \cdot x_i + b))$$

- If point is correctly classified and outside margin: Loss = 0
    
- If point is wrongly classified or inside margin: Loss > 0

## KERNELS (THE MAGIC TRICK)

When data cannot be separated by a straight line, SVM maps it to a higher dimension where it _can_ be separated linearly.

### **1. Linear Kernel**

Definition: No mapping, standard dot product.

Formula: $K(x, y) = x^T y$

Use When:

- Data is linearly separable.
    
- High-dimensional data (e.g., text classification) where number of features > number of samples.
    
- Fastest training time.
    

### **2. Polynomial Kernel**

Definition: Maps data to polynomial feature space.

Formula: $K(x, y) = (\gamma x^T y + r)^d$

Use When:

- Data has clear curved boundaries.
    
- Image processing.
    
- Risk: Can easily overfit if degree ($d$) is too high.
    

### **3. Radial Basis Function (RBF) Kernel**

Definition: Maps data to infinite-dimensional space based on distance.

Formula: $K(x, y) = \exp(-\gamma ||x - y||^2)$

Use When:

- **Default choice** for most non-linear problems.
    
- No prior knowledge of data distribution.
    
- Creates "islands" of decision boundaries around data clusters.
    

### **4. Sigmoid Kernel**

Definition: Similar to Neural Networks (tanh activation).

Formula: $K(x, y) = \tanh(\gamma x^T y + r)$

Use When:

- Specific proxy for neural networks.

## KEY HYPERPARAMETERS

### **1. C (Regularization Parameter)**

**Definition:** Controls the trade-off between achieving a low training error and a low testing error (Bias-Variance trade-off).

- **High C:**
    
    - Strict penalty for misclassification.
        
    - Tries to classify _all_ training points correctly.
        
    - **Result:** Small margin, complex boundary.
        
    - **Risk:** Overfitting (High Variance).
        
- **Low C:**
    
    - Loose penalty (tolerant of errors).
        
    - Accepts some misclassification for a wider margin.
        
    - **Result:** Wide margin, simple boundary.
        
    - **Risk:** Underfitting (High Bias).
        

### **2. Gamma (Kernel Coefficient)**

**Definition:** Defines how far the influence of a single training example reaches. (Used in RBF, Poly, Sigmoid).

- **High Gamma:**
    
    - "Close" reach. Only points very close to the hyperplane matter.
        
    - Model captures the shape of the data very tightly.
        
    - **Result:** Jagged, complex boundaries (Islands).
        
    - **Risk:** Overfitting.
        
- **Low Gamma:**
    
    - "Far" reach. Even distant points influence the decision.
        
    - **Result:** Smooth, linear-like boundaries.
        
    - **Risk:** Underfitting (too generalized).
        

### **3. kernel**

Definition: Specifies the kernel type to be used.

Values: 'linear', 'poly', 'rbf', 'sigmoid', 'precomputed'

Default: 'rbf'

### **4. degree**

Definition: Degree of the polynomial kernel function.

Default: 3

Impact: Higher degree = more flexible (wiggly) decision boundary but slower and prone to overfit.

## HYPERPARAMETER TUNING STRATEGY

**Start with:**

Python

```
# Baseline
kernel='rbf'
C=1.0
gamma='scale'
```

**If Underfitting (Model is too simple):**

- Increase `C` (make it stricter).
    
- Increase `gamma` (make it more sensitive to local data).
    
- Change kernel (if Linear is failing).
    

**If Overfitting (Model captures noise):**

- Decrease `C` (allow more errors/wider margin).
    
- Decrease `gamma` (smooth the boundary).
    
- Reduce polynomial `degree`.
    

**Grid Search Example:**

Python

```
param_grid = {
    'C': [0.1, 1, 10, 100],
    'gamma': [1, 0.1, 0.01, 0.001],
    'kernel': ['rbf', 'linear']
}
```

## ADVANTAGES OF SVM

1. **Effective in high dimensions** - Works well even when features > samples.
    
2. **Memory Efficient** - Uses only a subset of training points (support vectors).
    
3. **Versatile** - Different Kernel functions can be specified for decision function.
    
4. **Robust to outliers** - Especially in high-dimensional space (Soft margin absorbs them).
    
5. **Global Optimum** - Unlike Neural Networks (which can get stuck in local minima), the SVM optimization problem is convex (guaranteed global minimum).
    
6. **Good generalization** - Maximizing the margin usually prevents overfitting.
    

---

## DISADVANTAGES OF SVM

1. **Computationally Expensive** - Training time is $O(n^2)$ or $O(n^3)$. Very slow for large datasets ($>100k$ samples).
    
2. **Sensitive to Noise** - If classes overlap heavily, performance drops.
    
3. **No Probabilistic Interpretation** - Doesn't output probabilities directly (requires expensive 5-fold cross-validation calibration using Platt scaling).
    
4. **Feature Scaling Mandatory** - Sensitive to the scale of input data.
    
5. **Hard to Interpret** - Unlike Decision Trees, it is a "Black Box" model. Hard to explain _why_ a specific classification was made purely by looking at weights in RBF kernel.

## MULTICLASS CLASSIFICATION

SVM is natively a binary classifier. Two strategies are used for multiclass problems:

### **1. One-vs-Rest (OvR / OvA)**

- Split the multiclass problem into one binary problem per class.
    
- **Example:** Red vs [Blue, Green]; Blue vs [Red, Green]...
    
- Most common approach (default in `LinearSVC`).
    

### **2. One-vs-One (OvO)**

- Train a classifier for every pair of classes.
    
- **Example:** Red vs Blue, Red vs Green, Blue vs Green.
    
- For $k$ classes, you train $k(k-1)/2$ classifiers.
    
- Better for smaller datasets, but computationally heavy. Default in `SVC`.

## DATA PREPROCESSING FOR SVM

**Scaling is Non-Negotiable:**

SVM tries to maximize the distance between points. If one feature has a range [0, 1000] and another [0, 1], the distance will be dominated by the first feature.

**Correct Approach:**

Python

```
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

svm.fit(X_train_scaled, y_train)
```

Encoding:

Categorical variables must be OneHotEncoded or LabelEncoded.

## SVM VS OTHER ALGORITHMS

|**Aspect**|**SVM**|**Decision Trees**|**Logistic Regression**|**Random Forest**|
|---|---|---|---|---|
|**Boundaries**|Complex / Non-linear|Rectangular / Orthogonal|Linear|Complex / Jagged|
|**Outliers**|Robust (Soft Margin)|Sensitive|Sensitive|Robust|
|**Feature Scaling**|**Mandatory**|Not Required|Required|Not Required|
|**Interpretability**|Low (Black Box)|High (White Box)|High (Coefficients)|Low|
|**Speed (Train)**|Slow (Large Data)|Fast|Fast|Moderate|
|**Speed (Predict)**|Moderate|Very Fast|Very Fast|Fast|
|**Data Size**|Small/Medium|Any|Any|Any|

## TIME COMPLEXITY

**Training Complexity:**

```
O(n_samples² × n_features) to O(n_samples³ × n_features)

Depends heavily on cache size and dataset sparsity.
Not scalable for n_samples > 100,000.
```

**Prediction Complexity:**

```
O(n_features × n_support_vectors)

Prediction depends on number of support vectors, not total training data.
```

---

## BEST PRACTICES

1. **Always Scale Data:** Use `StandardScaler` or `MinMaxScaler`.
    
2. **Check Class Imbalance:** Use `class_weight='balanced'` if classes are uneven.
    
3. **Start with RBF:** It's the most versatile kernel.
    
4. **Tune C and Gamma together:** They interact closely. Use GridSearch.
    
5. **Use LinearSVC for large text data:** It's much faster than `SVC(kernel='linear')`.
    
6. **Visualize 2D slices:** To understand the decision boundary if possible.
