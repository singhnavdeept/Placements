

## EXAMPLE 1: WEATHER PREDICTION (SIMPLE CLASSIFICATION)

Let's predict whether to **Play Tennis** based on weather conditions.

### **Dataset:**

|Day|Outlook|Temperature|Humidity|Wind|Play Tennis|
|---|---|---|---|---|---|
|1|Sunny|Hot|High|Weak|No|
|2|Sunny|Hot|High|Strong|No|
|3|Overcast|Hot|High|Weak|Yes|
|4|Rain|Mild|High|Weak|Yes|
|5|Rain|Cool|Normal|Weak|Yes|
|6|Rain|Cool|Normal|Strong|No|
|7|Overcast|Cool|Normal|Strong|Yes|
|8|Sunny|Mild|High|Weak|No|
|9|Sunny|Cool|Normal|Weak|Yes|
|10|Rain|Mild|Normal|Weak|Yes|
|11|Sunny|Mild|Normal|Strong|Yes|
|12|Overcast|Mild|High|Strong|Yes|
|13|Overcast|Hot|Normal|Weak|Yes|
|14|Rain|Mild|High|Strong|No|

---

### **STEP-BY-STEP CALCULATION**

**Question:** Will we play tennis if:

- Outlook = Sunny
- Temperature = Cool
- Humidity = High
- Wind = Strong

---

### **Step 1: Calculate Prior Probabilities**

Count occurrences of each class:

```
P(Play = Yes) = 9/14 = 0.643
P(Play = No)  = 5/14 = 0.357
```

---

### **Step 2: Calculate Likelihoods for Each Feature**

**For Play = Yes (9 instances):**

**Outlook:**

```
P(Outlook=Sunny | Yes)    = 2/9 = 0.222
P(Outlook=Overcast | Yes) = 4/9 = 0.444
P(Outlook=Rain | Yes)     = 3/9 = 0.333
```

**Temperature:**

```
P(Temperature=Hot | Yes)  = 2/9 = 0.222
P(Temperature=Mild | Yes) = 4/9 = 0.444
P(Temperature=Cool | Yes) = 3/9 = 0.333
```

**Humidity:**

```
P(Humidity=High | Yes)    = 3/9 = 0.333
P(Humidity=Normal | Yes)  = 6/9 = 0.667
```

**Wind:**

```
P(Wind=Weak | Yes)        = 6/9 = 0.667
P(Wind=Strong | Yes)      = 3/9 = 0.333
```

---

**For Play = No (5 instances):**

**Outlook:**

```
P(Outlook=Sunny | No)     = 3/5 = 0.600
P(Outlook=Overcast | No)  = 0/5 = 0.000  ⚠️ Zero probability!
P(Outlook=Rain | No)      = 2/5 = 0.400
```

**Temperature:**

```
P(Temperature=Hot | No)   = 2/5 = 0.400
P(Temperature=Mild | No)  = 2/5 = 0.400
P(Temperature=Cool | No)  = 1/5 = 0.200
```

**Humidity:**

```
P(Humidity=High | No)     = 4/5 = 0.800
P(Humidity=Normal | No)   = 1/5 = 0.200
```

**Wind:**

```
P(Wind=Weak | No)         = 2/5 = 0.400
P(Wind=Strong | No)       = 3/5 = 0.600
```

---

### **Step 3: Apply Laplace Smoothing (α = 1)**

To avoid zero probabilities, add 1 to numerator and number of unique values to denominator.

**For Outlook (3 unique values: Sunny, Overcast, Rain):**

```
P(Outlook=Overcast | No) = (0 + 1) / (5 + 3) = 1/8 = 0.125
```

**All smoothed probabilities for No:**

```
Outlook:
P(Sunny | No)    = (3+1)/(5+3) = 4/8 = 0.500
P(Overcast | No) = (0+1)/(5+3) = 1/8 = 0.125
P(Rain | No)     = (2+1)/(5+3) = 3/8 = 0.375

Temperature:
P(Cool | No)     = (1+1)/(5+3) = 2/8 = 0.250

Humidity:
P(High | No)     = (4+1)/(5+2) = 5/7 = 0.714

Wind:
P(Strong | No)   = (3+1)/(5+2) = 4/7 = 0.571
```

---

### **Step 4: Calculate Posterior Probabilities**

**Test instance:** Outlook=Sunny, Temperature=Cool, Humidity=High, Wind=Strong

**For Play = Yes:**

```
P(Yes | features) ∝ P(Yes) × P(Sunny|Yes) × P(Cool|Yes) × P(High|Yes) × P(Strong|Yes)

= 0.643 × 0.222 × 0.333 × 0.333 × 0.333
= 0.643 × 0.00817
= 0.00525
```

**For Play = No:**

```
P(No | features) ∝ P(No) × P(Sunny|No) × P(Cool|No) × P(High|No) × P(Strong|No)

= 0.357 × 0.500 × 0.250 × 0.714 × 0.571
= 0.357 × 0.0510
= 0.0182
```

---

### **Step 5: Make Prediction**

```
P(Yes | features) = 0.00525
P(No | features)  = 0.0182

Since 0.0182 > 0.00525
Prediction: Play = No (Don't play tennis)
```

**Interpretation:** Given sunny weather with high humidity and strong wind, the model predicts we should NOT play tennis.

## EXAMPLE 2: FRUIT CLASSIFICATION (WITH NUMBERS)

Let's classify fruits based on their physical characteristics.

### **Dataset:**

|Fruit|Weight (g)|Color|Shape|Taste|
|---|---|---|---|---|
|Apple|150|Red|Round|Sweet|
|Apple|160|Red|Round|Sweet|
|Apple|140|Green|Round|Sour|
|Orange|130|Orange|Round|Sweet|
|Orange|140|Orange|Round|Sweet|
|Lemon|50|Yellow|Oval|Sour|
|Lemon|60|Yellow|Oval|Sour|
|Lemon|55|Yellow|Oval|Sour|

**Target:** Fruit (Apple, Orange, or Lemon)

---

### **MANUAL CALCULATION**

**Question:** What fruit is this?

- Weight = 145g
- Color = Red
- Shape = Round
- Taste = Sweet

---

### **Step 1: Prior Probabilities**

```
Total fruits = 8

P(Apple)  = 3/8 = 0.375
P(Orange) = 2/8 = 0.250
P(Lemon)  = 3/8 = 0.375
```

---

### **Step 2: Likelihoods**

**For Weight (continuous - use Gaussian NB):**

Calculate mean (μ) and standard deviation (σ) for each class:

```
Apple weights:  [150, 160, 140]
μ_apple = 150, σ_apple = 10

Orange weights: [130, 140]
μ_orange = 135, σ_orange = 7.07

Lemon weights:  [50, 60, 55]
μ_lemon = 55, σ_lemon = 5
```

**Gaussian probability for Weight=145:**

```
For Apple:
P(Weight=145 | Apple) = (1/√(2π×10²)) × exp(-(145-150)²/(2×10²))
                      = 0.0352

For Orange:
P(Weight=145 | Orange) = (1/√(2π×7.07²)) × exp(-(145-135)²/(2×7.07²))
                       = 0.0079

For Lemon:
P(Weight=145 | Lemon) = (1/√(2π×5²)) × exp(-(145-55)²/(2×5²))
                      ≈ 0 (very small)
```

---

**For Color (categorical):**

```
Apple (3 instances):
P(Red | Apple)    = 2/3 = 0.667
P(Green | Apple)  = 1/3 = 0.333

Orange (2 instances):
P(Orange | Orange) = 2/2 = 1.000

Lemon (3 instances):
P(Yellow | Lemon)  = 3/3 = 1.000
```

---

**For Shape (categorical):**

```
Apple:
P(Round | Apple) = 3/3 = 1.000

Orange:
P(Round | Orange) = 2/2 = 1.000

Lemon:
P(Oval | Lemon) = 3/3 = 1.000
```

---

**For Taste (categorical):**

```
Apple:
P(Sweet | Apple) = 2/3 = 0.667
P(Sour | Apple)  = 1/3 = 0.333

Orange:
P(Sweet | Orange) = 2/2 = 1.000

Lemon:
P(Sour | Lemon) = 3/3 = 1.000
```

---

### **Step 3: Calculate Posteriors**

**Test:** Weight=145, Color=Red, Shape=Round, Taste=Sweet

**For Apple:**

```
P(Apple | features) ∝ P(Apple) × P(145|Apple) × P(Red|Apple) × P(Round|Apple) × P(Sweet|Apple)

= 0.375 × 0.0352 × 0.667 × 1.000 × 0.667
= 0.00587
```

**For Orange:**

```
P(Orange | features) ∝ 0.250 × 0.0079 × 0 × 1.000 × 1.000
                      = 0  (Red never appears in Orange training data)
```

**For Lemon:**

```
P(Lemon | features) ∝ 0.375 × ~0 × 0 × 0 × 0
                    ≈ 0  (weight too high, wrong color, shape, taste)
```

---

### **Step 4: Prediction**

```
P(Apple | features)  = 0.00587
P(Orange | features) = 0
P(Lemon | features)  ≈ 0

Prediction: APPLE ✓
```

## EXAMPLE 3: EMAIL SPAM DETECTION (TEXT)

Simple text classification example.

### **Training Data:**

|Email|Text|Label|
|---|---|---|
|1|"buy cheap watches now"|Spam|
|2|"meeting tomorrow at 3pm"|Not Spam|
|3|"buy cheap pills online"|Spam|
|4|"project deadline next week"|Not Spam|
|5|"win lottery buy tickets"|Spam|
|6|"lunch meeting confirmed"|Not Spam|

**Vocabulary:** {buy, cheap, watches, now, meeting, tomorrow, at, 3pm, pills, online, project, deadline, next, week, win, lottery, tickets, lunch, confirmed}

---

### **Step 1: Build Word Count Matrix**

**Spam emails (3):**

```
Email 1: "buy cheap watches now"
Email 3: "buy cheap pills online"
Email 5: "win lottery buy tickets"

Word counts in Spam:
buy: 3
cheap: 2
watches: 1
now: 1
pills: 1
online: 1
win: 1
lottery: 1
tickets: 1
(all other words: 0)
```

**Not Spam emails (3):**

```
Email 2: "meeting tomorrow at 3pm"
Email 4: "project deadline next week"
Email 6: "lunch meeting confirmed"

Word counts in Not Spam:
meeting: 2
tomorrow: 1
at: 1
3pm: 1
project: 1
deadline: 1
next: 1
week: 1
lunch: 1
confirmed: 1
(all other words: 0)
```

---

### **Step 2: Calculate Probabilities (Multinomial NB)**

**Prior probabilities:**

```
P(Spam) = 3/6 = 0.5
P(Not Spam) = 3/6 = 0.5
```

**Word probabilities with Laplace smoothing (α=1):**

Total unique words (V) = 19

**For Spam (total words = 12):**

```
P(buy | Spam) = (3 + 1) / (12 + 19) = 4/31 = 0.129
P(cheap | Spam) = (2 + 1) / (12 + 19) = 3/31 = 0.097
P(meeting | Spam) = (0 + 1) / (12 + 19) = 1/31 = 0.032
P(project | Spam) = (0 + 1) / (12 + 19) = 1/31 = 0.032
```

**For Not Spam (total words = 11):**

```
P(buy | Not Spam) = (0 + 1) / (11 + 19) = 1/30 = 0.033
P(cheap | Not Spam) = (0 + 1) / (11 + 19) = 1/30 = 0.033
P(meeting | Not Spam) = (2 + 1) / (11 + 19) = 3/30 = 0.100
P(project | Not Spam) = (1 + 1) / (11 + 19) = 2/30 = 0.067
```

---

### **Step 3: Classify New Email**

**Test email:** "buy cheap meeting"

Words: [buy, cheap, meeting]

**For Spam:**

```
P(Spam | "buy cheap meeting") ∝ P(Spam) × P(buy|Spam) × P(cheap|Spam) × P(meeting|Spam)

= 0.5 × 0.129 × 0.097 × 0.032
= 0.5 × 0.000400
= 0.000200
```

**For Not Spam:**

```
P(Not Spam | "buy cheap meeting") ∝ 0.5 × 0.033 × 0.033 × 0.100

= 0.5 × 0.000109
= 0.0000545
```

**Prediction:**

```
0.000200 > 0.0000545
Prediction: SPAM ✓
```

**Why?** Words "buy" and "cheap" appear much more frequently in spam emails.

## EXAMPLE 4: STUDENT PASS/FAIL PREDICTION

Predict if a student will pass based on study hours and attendance.

### **Dataset:**

|Student|Study Hours|Attendance (%)|Previous Grade|Pass/Fail|
|---|---|---|---|---|
|1|2|60|C|Fail|
|2|8|95|A|Pass|
|3|6|80|B|Pass|
|4|1|50|D|Fail|
|5|7|90|A|Pass|
|6|3|65|C|Fail|
|7|9|98|A|Pass|
|8|5|75|B|Pass|

---

### **Using Gaussian Naive Bayes**

**Step 1: Separate by class**

**Pass (5 students):**

```
Study Hours: [8, 6, 7, 9, 5]
Attendance: [95, 80, 90, 98, 75]
Previous Grade: [A, B, A, A, B]
```

**Fail (3 students):**

```
Study Hours: [2, 1, 3]
Attendance: [60, 50, 65]
Previous Grade: [C, D, C]
```

---

**Step 2: Calculate statistics**

**For Study Hours:**

```
Pass:  μ = 7,   σ = 1.58
Fail:  μ = 2,   σ = 1.0
```

**For Attendance:**

```
Pass:  μ = 87.6, σ = 9.76
Fail:  μ = 58.3, σ = 7.64
```

**For Previous Grade (categorical):**

```
Pass:
P(A | Pass) = 3/5 = 0.60
P(B | Pass) = 2/5 = 0.40

Fail:
P(C | Fail) = 2/3 = 0.67
P(D | Fail) = 1/3 = 0.33
```

---

**Step 3: Predict for new student**

**Test:** Study Hours=6, Attendance=85%, Previous Grade=B

**Prior:**

```
P(Pass) = 5/8 = 0.625
P(Fail) = 3/8 = 0.375
```

**Likelihoods:**

**For Pass:**

```
P(Study=6 | Pass) ≈ 0.23  (from Gaussian formula)
P(Attendance=85 | Pass) ≈ 0.04
P(Grade=B | Pass) = 0.40

P(Pass | features) ∝ 0.625 × 0.23 × 0.04 × 0.40 = 0.0023
```

**For Fail:**

```
P(Study=6 | Fail) ≈ 0.00004  (very unlikely - too high)
P(Attendance=85 | Fail) ≈ 0.00007  (very unlikely - too high)
P(Grade=B | Fail) = 0 (with smoothing: very small)

P(Fail | features) ∝ 0.375 × 0.00004 × 0.00007 × small ≈ 0
```

**Prediction:** PASS ✓

**Interpretation:** Student studies 6 hours (above fail average of 2), has 85% attendance (above fail average of 58%), and got grade B previously - all indicators point to passing.

## KEY INSIGHTS FROM EXAMPLES

### **1. The "Naive" Assumption in Action**

In Weather example:

- We multiply probabilities independently
- Reality: High humidity might be correlated with certain outlooks
- But algorithm treats them separately
- Still makes good predictions!

### **2. Zero Probability Problem**

In Weather example:

- "Overcast" never appeared in "No" class
- Without smoothing: P(Overcast|No) = 0
- This would make P(No|Overcast,...) = 0 always
- Smoothing fixes this: P(Overcast|No) = 1/8

### **3. Why It Works Despite Being "Naive"**

In Spam example:

- "buy" and "cheap" often appear together (correlated)
- Algorithm ignores correlation
- But both words independently suggest spam
- Multiplying their probabilities still gives correct ranking

### **4. Small Probabilities**

Notice all posterior probabilities are very small:

- 0.00525, 0.0182, etc.
- **We only care about which is larger**, not absolute values
- Naive Bayes ranks classes correctly

### **5. Different Features, Same Framework**

- Weather: All categorical → Count frequencies
- Fruit: Mixed (numerical + categorical) → Gaussian + counting
- Spam: Text → Word counts (Multinomial)
- Student: Continuous → Gaussian distributions

**Same algorithm, different probability calculations!**

## VISUALIZATION OF DECISION BOUNDARIES

### **Simple 2D Example:**

```
Dataset:
Class A (circles): Points near (2, 2), (3, 3), (2.5, 2.5)
Class B (squares): Points near (7, 7), (8, 8), (7.5, 7.5)

Naive Bayes Decision:
- Calculates P(A|x,y) and P(B|x,y)
- Decision boundary roughly at x+y = 10
- New point (5, 5): Predicts Class A
- New point (9, 9): Predicts Class B
```

**Mental Model:**

- Naive Bayes creates probability "clouds" around each class
- New point: Which cloud is it more likely from?
- Clouds overlap: predict based on probabilities

## PRACTICAL TAKEAWAYS

1. **Probabilities multiply** → Numbers get very small
    
    - Normal! Just compare which is larger
2. **Smoothing is essential** → Prevents zeros
    
    - Always use α ≥ 1
3. **Independence assumption** → Features treated separately
    
    - Naive but effective
4. **Right variant matters** → Match data type
    
    - Continuous → Gaussian
    - Counts → Multinomial
    - Binary → Bernoulli
5. **Fast and simple** → Great baseline
    
    - Quick to implement and test
    - Often "good enough"

These simple examples show the core intuition: Naive Bayes calculates "How likely is this data point from each class?" and picks the most likely class.
