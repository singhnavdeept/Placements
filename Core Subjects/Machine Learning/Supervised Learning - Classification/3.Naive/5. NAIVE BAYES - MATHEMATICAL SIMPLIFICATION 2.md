That mathematical expansion is known as the **Conditional Chain Rule** (or the **General Multiplication Rule** for conditional probability).

It is the **unsimplified** way to express the **Likelihood** term, $P(X|y)$, in Bayes' Theorem when the features are **dependent** on each other.

---

## 🔗 The Conditional Chain Rule Explained

The expression shows how to calculate the **joint conditional probability** of an entire set of features, $X = (x_1, x_2, \dots, x_n)$, occurring, given a specific class $y$. It breaks down the single complex joint probability into a sequence of simpler, chained conditional probabilities.

$$P(X|y) = P(x_1|y) \times P(x_2|y, x_1) \times P(x_3|y, x_1, x_2) \times \dots \times P(x_n|y, x_1, \dots, x_{n-1})$$

### 1. The Starting Term: $P(x_1|y)$

The first term, $P(x_1|y)$, is the probability of the first feature, $x_1$, occurring, given only the class $y$. This is a straightforward conditional probability.

### 2. The Subsequent Terms: $P(x_i | y, x_1, \dots, x_{i-1})$

Each subsequent term calculates the conditional probability of the next feature, $x_i$, occurring. Crucially, it must be conditioned not only on the class $y$ but also on **all of the features that have already been accounted for** ($x_1$ up to $x_{i-1}$).

- **For $x_2$**: $P(x_2|y, x_1)$ is the probability of $x_2$ given the class $y$ **and** that $x_1$ has already occurred.
    
- **For $x_3$**: $P(x_3|y, x_1, x_2)$ is the probability of $x_3$ given the class $y$ **and** that both $x_1$ and $x_2$ have already occurred.
    
- **For $x_n$**: $P(x_n|y, x_1, \dots, x_{n-1})$ is the probability of the last feature $x_n$ given the class $y$ **and** all preceding features.
    

### The Problem in Practice

This formula is **mathematically exact** for the Likelihood term. However, for a real-world machine learning problem, especially one with a large number of features ($n$), it is practically **unusable**:

- **Data Scarcity**: To estimate the term $P(x_n|y, x_1, \dots, x_{n-1})$ accurately, you would need to have observed a sufficient number of training samples that match the exact combination of $(y, x_1, \dots, x_{n-1})$. With many features, the number of possible combinations grows exponentially, leading to zero counts for most combinations in the training data.
    
- **Computational Cost**: Storing and calculating all these highly specific conditional probabilities is computationally infeasible.
    

## 💡 The Naive Bayes Simplification

This is where the **Naive Bayes assumption** steps in. By assuming that all features are **conditionally independent** given the class $y$, we state that knowledge of $x_1, x_2, \dots, x_{i-1}$ adds no additional information about $x_i$ once the class $y$ is known.

Mathematically, the assumption allows us to simplify every single dependent term:

$$P(x_i|y, x_1, \dots, x_{i-1}) \approx P(x_i|y)$$

The complex chain rule expression then simplifies into the basic multiplicative form used in the Naive Bayes model:

$$P(X|y) = P(x_1|y) \times P(x_2|y) \times P(x_3|y) \times \dots \times P(x_n|y)$$
