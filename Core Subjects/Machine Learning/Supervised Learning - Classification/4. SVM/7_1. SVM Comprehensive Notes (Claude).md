## SUPPORT VECTOR MACHINES (SVM)

**Definition:** A supervised learning algorithm that finds the optimal hyperplane (decision boundary) that maximally separates different classes in the feature space. It works by identifying the support vectors (data points closest to the decision boundary) and maximizing the margin (distance) between classes. Can handle linear and non-linear classification/regression through the use of kernel functions.

**Used For:**

- Binary classification (primary use)
- Multi-class classification (one-vs-one or one-vs-all)
- Regression (SVR - Support Vector Regression)
- Outlier detection (One-Class SVM)
- Text classification
- Image classification
- Bioinformatics (protein classification, gene expression)

**Cases of Application:**

- Face detection and recognition
- Text and document categorization
- Handwriting recognition
- Image classification
- Bioinformatics (cancer classification, protein structure prediction)
- Stock market prediction
- Sentiment analysis
- Spam filtering
- Medical diagnosis
- Credit scoring
- Speech recognition
- Fraud detection

**Prerequisites:**

- Labeled training data
- Features should be numerical
- **Feature scaling is CRITICAL** (StandardScaler)
- Handle missing values before training
- Consider dimensionality reduction for high-dimensional data
- Sufficient memory for kernel matrix (can be O(n²))
- Balanced classes preferred (or use class weights)
- Computational resources for training (can be slow on large datasets)

**Performance Measures:**

- **Classification:** Accuracy, Precision, Recall, F1-Score, Confusion Matrix, ROC-AUC, Log Loss
- **Regression:** MAE, MSE, RMSE, R² Score
- **SVM-specific:**
    - Number of support vectors (fewer is better)
    - Margin width (larger is better)
    - Training time
    - Support vector ratio (support vectors / total samples)
- Cross-validation scores
- Grid search for hyperparameter tuning

## HOW SVM WORKS

### **Core Concept:**

Find the hyperplane that **maximally separates** classes while **maximizing the margin**.

**Visual Intuition (2D):**

```
Class A (•)    Decision Boundary (|)    Class B (×)

    •              |                   ×
      •            |                 ×
        •          |               ×
      •   •        |             ×   ×
    •       •      |           ×       ×
        •          |               ×
      •            |                 ×

    <----margin---->
    
Support vectors: Points closest to boundary (circled)
Margin: Distance between classes
Goal: Maximize margin
```

### **Key Ideas:**

1. **Hyperplane:** Decision boundary that separates classes
    
    - In 2D: Line
    - In 3D: Plane
    - In n-D: (n-1)-dimensional hyperplane
2. **Support Vectors:** Data points closest to the hyperplane
    
    - Only these points matter for the decision boundary
    - Other points can be removed without affecting the model
    - Typically only 20-60% of training data
3. **Margin:** Distance between hyperplane and nearest points of each class
    
    - Larger margin = better generalization
    - SVM maximizes this margin
4. **Hard Margin vs Soft Margin:**
    
    - **Hard Margin:** Perfect separation, no misclassification allowed
    - **Soft Margin:** Allows some misclassification (more practical)

## MATHEMATICAL FOUNDATION

### **Linear SVM (Separable Case)**

**Hyperplane Equation:**

```
w·x + b = 0

Where:
- w = weight vector (perpendicular to hyperplane)
- x = input features
- b = bias term (intercept)
```

**Decision Function:**

```
f(x) = sign(w·x + b)

If w·x + b > 0  → Class +1
If w·x + b < 0  → Class -1
If w·x + b = 0  → On the hyperplane
```

**Margin:**

```
Margin = 2 / ||w||

Where ||w|| = magnitude of weight vector
```

**Optimization Goal:**

```
Maximize: 2 / ||w||

Equivalent to:
Minimize: (1/2) ||w||²

Subject to: yᵢ(w·xᵢ + b) ≥ 1  for all i

Where:
- yᵢ ∈ {-1, +1} (class labels)
- Constraint ensures correct classification with margin ≥ 1
```

---

### **Soft Margin SVM (Non-Separable Case)**

**Problem:** Real data often not perfectly linearly separable.

**Solution:** Allow some misclassifications using **slack variables** (ξ).

**Modified Optimization:**

```
Minimize: (1/2) ||w||² + C Σξᵢ

Subject to: 
- yᵢ(w·xᵢ + b) ≥ 1 - ξᵢ  for all i
- ξᵢ ≥ 0

Where:
- ξᵢ = slack variable (allows violation of margin)
- C = regularization parameter (controls trade-off)
```

**Slack Variable (ξ) Interpretation:**

```
ξᵢ = 0           → Point correctly classified outside margin
0 < ξᵢ < 1       → Point inside margin but correctly classified
ξᵢ = 1           → Point on decision boundary
ξᵢ > 1           → Point misclassified
```

**C Parameter:**

- **Large C:** Small margin, fewer misclassifications (hard margin, overfitting risk)
- **Small C:** Large margin, more misclassifications (soft margin, underfitting risk)

## THE KERNEL TRICK

**Problem:** Many real-world problems are not linearly separable.

**Solution:** Map data to higher-dimensional space where it becomes linearly separable.

**Example (1D → 2D):**

```
Original (1D): Cannot separate
x: -2, -1, 1, 2
y:  -, -, +, +

    - - | + +  ← No line separates perfectly
    
Transform: x → (x, x²)
New space (2D):
(-2, 4)  Class -
(-1, 1)  Class -
(1, 1)   Class +
(2, 4)   Class +

Now linearly separable in 2D!
```

**Kernel Trick:** Instead of explicitly computing the transformation φ(x), compute inner products in high-dimensional space directly using kernel function K(x, x').

```
K(x, x') = φ(x) · φ(x')

Advantage: Never compute φ(x) explicitly (efficient!)
```

## KERNEL FUNCTIONS

### **1. LINEAR KERNEL (No Transformation)**

**Formula:**

```
K(x, x') = x · x'
```

**Properties:**

- Equivalent to standard linear SVM
- No transformation to higher dimension
- Fastest computation
- Best for linearly separable data

**Use When:**

- Data is linearly separable
- High-dimensional data (text, sparse features)
- Large datasets (fastest training)
- Want interpretability

**Avoid When:**

- Data clearly non-linear
- Need complex decision boundaries

**Example:**

```python
from sklearn.svm import SVC
model = SVC(kernel='linear', C=1.0)
```

---

### **2. POLYNOMIAL KERNEL**

**Formula:**

```
K(x, x') = (γ(x · x') + r)^d

Where:
- d = degree of polynomial
- γ (gamma) = coefficient
- r = independent term (coef0)
```

**Common Values:**

- d = 2 (quadratic, default for testing)
- d = 3 (cubic, common choice)
- d = 4+ (rarely used, overfitting risk)
- γ = 1/n_features (default)
- r = 0 (default)

**Properties:**

- Creates polynomial decision boundaries
- Degree controls complexity
- Can model curved boundaries
- More flexible than linear

**Decision Boundary Examples:**

```
d = 2: Circles, ellipses, parabolas
d = 3: More complex curves
d = 4+: Very complex (overfitting risk)
```

**Use When:**

- Data has polynomial relationships
- Need curved decision boundaries
- Moderate feature dimensions
- Know degree of non-linearity

**Avoid When:**

- Don't know degree of polynomial needed
- High-dimensional data (computationally expensive)
- Want to avoid overfitting
- RBF works just as well (try RBF first)

**Example:**

```python
model = SVC(kernel='poly', degree=3, gamma='scale', coef0=0)
```

---

### **3. RBF KERNEL (Radial Basis Function / Gaussian Kernel)**

**Formula:**

```
K(x, x') = exp(-γ ||x - x'||²)

Where:
- γ (gamma) = 1 / (2σ²)
- σ = standard deviation
- ||x - x'|| = Euclidean distance
```

**Properties:**

- **Most popular kernel** (default in sklearn)
- Maps to infinite-dimensional space
- Creates circular/elliptical decision boundaries
- Very flexible, can approximate any decision boundary
- Universal approximator
- Smooth decision boundaries

**Gamma (γ) Parameter:**

**Large γ (e.g., 10):**

- Narrow influence (only nearby points matter)
- Complex, wiggly decision boundaries
- High variance, low bias
- **Overfitting risk**
- Each support vector has small influence radius

**Small γ (e.g., 0.01):**

- Wide influence (far points matter)
- Smooth, simple decision boundaries
- Low variance, high bias
- **Underfitting risk**
- Each support vector has large influence radius

**Visual:**

```
Large γ:  )))  Tight boundary around points
Small γ:  (((((  Smooth, wide boundary
```

**Default Values:**

- 'scale': γ = 1 / (n_features × var(X))
- 'auto': γ = 1 / n_features

**Use When:**

- Default choice for most problems
- Don't know the relationship between features
- Need flexible decision boundaries
- Have enough data to tune γ properly

**Avoid When:**

- Very high-dimensional sparse data (linear might be better)
- Large datasets (computationally expensive)
- Need interpretability (use linear)

**Example:**

```python
model = SVC(kernel='rbf', gamma='scale', C=1.0)
```

---

### **4. SIGMOID KERNEL (Tanh Kernel)**

**Formula:**

```
K(x, x') = tanh(γ(x · x') + r)

Similar to neural network activation
```

**Properties:**

- Similar to neural networks
- Can act as 2-layer perceptron
- Rarely used in practice
- Can have negative values (not a true kernel in all cases)

**Use When:**

- Mimicking neural network behavior
- Specific theoretical reasons
- Historical compatibility

**Avoid When:**

- In most practical cases (use RBF instead)

**Example:**

```python
model = SVC(kernel='sigmoid', gamma='scale', coef0=0)
```

---

### **5. CUSTOM KERNELS**

**Definition:** User-defined kernel functions.

**Requirements:**

- Must satisfy Mercer's condition (positive semi-definite)
- Returns similarity matrix

**Example:**

```python
def custom_kernel(X, Y):
    # Your custom similarity function
    return np.dot(X, Y.T)

model = SVC(kernel=custom_kernel)
```

**Use When:**

- Domain-specific similarity measures
- Special data types (graphs, strings, sequences)
- Research/experimental purposes

---

### **KERNEL COMPARISON:**

|Kernel|Complexity|Speed|Decision Boundary|Best For|Default Params|
|---|---|---|---|---|---|
|**Linear**|Low|Fast|Linear|High-dim data, text|C=1.0|
|**Polynomial**|Medium|Medium|Curves (degree d)|Known polynomial|degree=3, gamma='scale'|
|**RBF**|High|Slow|Flexible, smooth|**Most problems**|gamma='scale', C=1.0|
|**Sigmoid**|Medium|Medium|S-shaped|Rare|gamma='scale'|

**General Rule:** Start with RBF, try Linear if RBF too slow or data high-dimensional.

## KEY HYPERPARAMETERS

### **1. C (Regularization Parameter)**

**Definition:** Controls trade-off between maximizing margin and minimizing classification error.

**Range:** (0, ∞), typically [0.01, 100]

**Default:** 1.0

**Impact:**

**Large C (e.g., 100):**

- Small margin, fewer errors on training data
- Hard margin approach
- Low bias, high variance
- **Overfitting risk**
- More support vectors
- Complex decision boundary

**Small C (e.g., 0.01):**

- Large margin, more errors allowed
- Soft margin approach
- High bias, low variance
- **Underfitting risk**
- Fewer support vectors
- Simple decision boundary

**Visual:**

```
C = 0.01 (Soft Margin):
  •  •  |  ×  ×    ← Some points allowed inside margin
    •   |   ×
  •     |     ×
  
C = 100 (Hard Margin):
  •  • || ×  ×     ← Tight fit, no violations
    •  ||  ×
  •    ||    ×
```

**Tuning Strategy:**

- Start with C=1.0
- If overfitting: decrease C
- If underfitting: increase C
- Use grid search: [0.01, 0.1, 1, 10, 100]

**Relationship with Overfitting:**

```
Overfitting:  Training acc = 99%, Test acc = 70%  → Decrease C
Underfitting: Training acc = 65%, Test acc = 63%  → Increase C
Good fit:     Training acc = 85%, Test acc = 83%  → C is good
```

---

### **2. gamma (γ) - For RBF, Polynomial, Sigmoid Kernels**

**Definition:** Defines influence radius of support vectors. Controls how far the influence of a single training example reaches.

**Range:** (0, ∞), typically [0.0001, 1]

**Default:** 'scale' = 1 / (n_features × var(X))

**Values:**

- 'scale': Recommended default
- 'auto': 1 / n_features
- float: Specific value

**Impact:**

**Large gamma (e.g., 10):**

- Small influence radius
- Decision boundary follows training data closely
- More complex boundary
- Low bias, high variance
- **Overfitting risk**

**Small gamma (e.g., 0.001):**

- Large influence radius
- Smooth decision boundary
- Simpler boundary
- High bias, low variance
- **Underfitting risk**

**Analogy:**

```
Think of gamma as "influence radius" of each point:

High gamma (γ=10):
  Point influences only its immediate neighbors
  Like a spotlight with narrow beam
  
Low gamma (γ=0.01):
  Point influences points far away
  Like a floodlight with wide beam
```

**Mathematical Interpretation:**

```
K(x, x') = exp(-γ ||x - x'||²)

When ||x - x'|| = 0 (same point): K = 1
When ||x - x'|| large: K → 0

- Large γ: K drops to 0 quickly (small radius)
- Small γ: K drops to 0 slowly (large radius)
```

**Tuning Strategy:**

- Use 'scale' or 'auto' as starting point
- If overfitting: decrease gamma
- If underfitting: increase gamma
- Grid search: [0.0001, 0.001, 0.01, 0.1, 1]

---

### **3. kernel**

**Definition:** Specifies kernel type to use.

**Values:**

- 'rbf': Radial Basis Function (default, recommended)
- 'linear': Linear kernel
- 'poly': Polynomial kernel
- 'sigmoid': Sigmoid kernel
- callable: Custom kernel function

**Decision Guide:**

```
START HERE:
├─ Is data linearly separable? → 'linear'
├─ High-dimensional sparse data (text)? → 'linear'
├─ Don't know? → 'rbf' (try this first!)
├─ Know polynomial relationship? → 'poly'
└─ Need neural network-like? → 'sigmoid' (rare)
```

---

### **4. degree (for Polynomial Kernel)**

**Definition:** Degree of polynomial kernel function.

**Range:** Integers ≥ 1

**Default:** 3

**Common Values:**

- 2: Quadratic (circles, ellipses)
- 3: Cubic (default, good balance)
- 4+: Rarely needed (overfitting risk)

**Impact:**

- Higher degree = more complex boundaries
- Degree 1 = linear kernel
- Higher degree = slower training

---

### **5. coef0 (for Polynomial and Sigmoid)**

**Definition:** Independent term in kernel function.

**Range:** Real numbers

**Default:** 0.0

**Impact:**

- Affects influence of higher-degree vs lower-degree terms
- Usually keep at 0.0 unless specific reason
- Rarely tuned in practice

---

### **6. class_weight**

**Definition:** Weights for classes.

**Values:**

- None: All classes equal weight (default)
- 'balanced': Automatically adjust weights inversely proportional to class frequencies
- dict: {class: weight}

**Formula for 'balanced':**

```
weight[class] = n_samples / (n_classes × n_samples_class)
```

**Use When:**

- Imbalanced datasets
- One class much rarer than others
- Cost of misclassifying minority class is high

**Example:**

```python
# Imbalanced: 900 class 0, 100 class 1
model = SVC(kernel='rbf', class_weight='balanced')

# Manual weights
model = SVC(class_weight={0: 1, 1: 9})
```

---

### **7. max_iter**

**Definition:** Maximum number of iterations for solver.

**Default:** -1 (no limit)

**Range:** -1 or positive integer

**Impact:**

- Controls training time
- -1: Train until convergence
- Positive int: Stop after max_iter iterations
- May not converge with low values

**Use When:**

- Need to limit training time
- Large datasets
- Debugging

---

### **8. random_state**

**Definition:** Seed for random number generator.

**Impact:**

- Controls randomness in data shuffling
- Ensures reproducibility
- Important for consistent results

---

### **9. probability**

**Definition:** Enable probability estimates.

**Values:**

- False (default): Faster, no probabilities
- True: Slower, enables predict_proba()

**Impact:**

- True: Fits additional logistic regression
- Adds computational cost
- Needed for predict_proba() and predict_log_proba()

**Use When:**

- Need probability estimates
- Want confidence scores
- Building ensemble models

---

### **10. cache_size**

**Definition:** Size of kernel cache in MB.

**Default:** 200 MB

**Impact:**

- Larger cache = faster training (more memory)
- Stores kernel computations
- Useful for large datasets

## HYPERPARAMETER TUNING STRATEGY

### **Step 1: Start with Defaults**

```python
model = SVC(kernel='rbf', C=1.0, gamma='scale')
```

### **Step 2: Coarse Grid Search**

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'C': [0.1, 1, 10, 100],
    'gamma': [0.001, 0.01, 0.1, 1],
    'kernel': ['rbf', 'linear']
}

grid = GridSearchCV(SVC(), param_grid, cv=5)
grid.fit(X_train, y_train)
print(grid.best_params_)
```

### **Step 3: Fine Grid Search**

```python
# Refine around best values
param_grid = {
    'C': [8, 9, 10, 11, 12],
    'gamma': [0.008, 0.009, 0.01, 0.011, 0.012]
}
```

### **Common Patterns:**

**If Overfitting:**

- Decrease C
- Decrease gamma (for RBF)
- Try simpler kernel (linear instead of RBF)
- Add more data
- Use cross-validation

**If Underfitting:**

- Increase C
- Increase gamma (for RBF)
- Try more complex kernel (RBF instead of linear)
- Add more features
- Check feature scaling

## MULTI-CLASS SVM

**Problem:** SVM is inherently binary classifier.

**Solutions:**

### **1. One-vs-Rest (OvR / OvA)**

**Strategy:** Train k binary classifiers (one per class).

- Class i vs all other classes
- Predict class with highest decision score

**Example (3 classes):**

```
Classifier 1: Class A vs (B, C)
Classifier 2: Class B vs (A, C)
Classifier 3: Class C vs (A, B)

Predict: Choose classifier with highest score
```

**Properties:**

- k classifiers needed
- Fast training (can parallelize)
- Default in sklearn
- Can have ambiguous regions

---

### **2. One-vs-One (OvO)**

**Strategy:** Train k(k-1)/2 binary classifiers (one per class pair).

- Each classifier: Class i vs Class j
- Predict by majority voting

**Example (3 classes):**

```
Classifier 1: A vs B
Classifier 2: A vs C
Classifier 3: B vs C

Predict: Class with most votes
```

**Properties:**

- k(k-1)/2 classifiers needed (more models)
- Each trained on smaller subset (faster per model)
- More accurate than OvR for some problems
- More robust to imbalanced data

---

### **In sklearn:**

```python
from sklearn.svm import SVC
from sklearn.multiclass import OneVsRestClassifier, OneVsOneClassifier

# Default (OvR)
model = SVC(kernel='rbf')  # Uses OvR internally

# Explicit OvR
model = OneVsRestClassifier(SVC(kernel='rbf'))

# Explicit OvO
model = OneVsOneClassifier(SVC(kernel='rbf'))
```

## SUPPORT VECTOR REGRESSION (SVR)

**Definition:** Extension of SVM to regression problems. Finds function that deviates from actual targets by at most ε (epsilon).

### **Key Differences from Classification:**

1. **Goal:** Fit data within ε-tube instead of separating classes
2. **Loss:** ε-insensitive loss (ignore errors < ε)
3. **Output:** Continuous values instead of class labels

### **ε-Tube Concept:**

```
Predicted Line
      |
   ε |========| ε  ← Tube of width 2ε
      |
      
Points inside tube: No penalty
Points outside tube: Penalized
```

### **Parameters:**

**epsilon (ε):**

- Width of ε-tube
- Errors < ε ignored
- Larger ε: simpler model, more points in tube
- Smaller ε: complex model, tighter fit

**C:** Same as classification (regularization)

**kernel:** Same options as classification

### **Example:**

```python
from sklearn.svm import SVR

model = SVR(kernel='rbf', C=1.0, epsilon=0.1, gamma='scale')
model.fit(X_train, y_train)
predictions = model.predict(X_test)
```

### **When to Use SVR:**

- Robust regression needed
- Outliers present
- Non-linear relationships
- High-dimensional data
- Care only about errors above threshold ε

## ADVANTAGES OF SVM

1. **Effective in high-dimensional spaces** - Works well with many features
2. **Memory efficient** - Only uses support vectors (subset of training data)
3. **Versatile** - Different kernels for different problems
4. **Robust to overfitting** - In high-dimensional space with proper regularization
5. **Works well with clear margin** - When classes well-separated
6. **Effective with fewer samples** - Can work with smaller datasets than neural networks
7. **Kernel trick** - Can model complex non-linear boundaries
8. **Global optimum** - Convex optimization (unlike neural networks)
9. **Theoretically well-founded** - Strong mathematical basis (VC theory)
10. **Binary classification excellence** - State-of-art for binary problems
11. **Handles non-linear relationships** - Through kernel functions
12. **Less prone to local minima** - Convex optimization problem

---

## DISADVANTAGES OF SVM

1. **Not suitable for large datasets** - Training time O(n²) to O(n³)
2. **Slow training** - Especially with large n and non-linear kernels
3. **Memory intensive** - Kernel matrix can be huge O(n²)
4. **Black box (non-linear)** - Hard to interpret with RBF/polynomial kernels
5. **Hyperparameter sensitivity** - Performance depends heavily on C, gamma
6. **No direct probability estimates** - Need additional calibration
7. **Sensitive to feature scaling** - MUST normalize/standardize features
8. **Choice of kernel is difficult** - No clear guide for kernel selection
9. **Doesn't work well with noisy data** - Overlapping classes problematic
10. **Multi-class classification inefficient** - Need multiple binary classifiers
11. **No built-in feature importance** - Can't easily identify important features
12. **Sensitive to outliers** - Support vectors can be outliers

## WHEN TO USE SVM

**Use SVM When:**

- Binary classification problem
- High-dimensional data (text classification, genomics)
- Clear margin of separation exists
- Small to medium-sized datasets (n < 10,000)
- Non-linear decision boundary needed (use RBF kernel)
- Robust to overfitting required
- Have resources for hyperparameter tuning
- Want theoretically sound model
- Data has fewer samples than features
- Outlier robustness with proper C value

**Avoid SVM When:**

- Very large datasets (n > 100,000) - use logistic regression, SGD
- Need very fast training time - use Naive Bayes, logistic regression
- Need probability estimates - use logistic regression (native probabilities)
- Need interpretability - use logistic regression, decision trees
- Multi-class with many classes - use Random Forest, neural networks
- Real-time prediction needed - use simpler models
- Cannot afford extensive hyperparameter tuning
- Computational resources limited
- Data has extreme noise or class overlap

## FEATURE SCALING - CRITICAL!

**Why SVM Needs Scaling:**

SVM uses distance-based calculations. Features with larger ranges dominate the distance metric.

**Example:**

```
Feature 1 (Age): 20-80 (range 60)
Feature 2 (Income): 20,000-100,000 (range 80,000)

Distance dominated by Income!
Age becomes irrelevant.
```

### **Scaling Methods:**

#### **1. StandardScaler (Standardization) - Recommended**

**Formula:**

```
x_scaled = (x - μ) / σ

Where:
- μ = mean
- σ = standard deviation
```

**Properties:**

- Mean = 0, Std = 1
- Preserves outliers
- Best for SVM with RBF kernel
- **Default choice**

**Code:**

```python
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)  # Use same scaler!

model = SVC(kernel='rbf')
model.fit(X_train_scaled, y_train)
```

#### **2. MinMaxScaler (Normalization)**

**Formula:**

```
x_scaled = (x - x_min) / (x_max - x_min)

Range: [0, 1]
```

**Properties:**

- Bounded range
- Preserves zero values
- Sensitive to outliers

#### **3. RobustScaler**

**Uses median and IQR instead of mean and std:**

- Robust to outliers
- Good for data with extreme outliers

### **CRITICAL RULES:**

1. **Always scale features for SVM** (except with linear kernel on normalized data)
2. **Fit scaler on training data only**
3. **Apply same scaler to test/validation data**
4. **Scale before train-test split? No! Scale after split**

**Common Mistake:**

```python
# WRONG - causes data leakage!
X_scaled = scaler.fit_transform(X)
X_train, X_test = train_test_split(X_scaled)

# CORRECT
X_train, X_test = train_test_split(X)
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)
```

## HANDLING IMBALANCED DATA

**Problem:** One class has many more samples than others.

### **Solutions:**

#### **1. Class Weights (Best for SVM)**

```python
model = SVC(kernel='rbf', class_weight='balanced')

# Or manual
model = SVC(class_weight={0: 1, 1: 10})
```

**How it works:** Adjusts C parameter for each class.

---

#### **2. Resampling**

**Oversample minority class:**

```python
from imblearn.over_sampling import SMOTE

smote = SMOTE()
```

## LINEAR SVM VS KERNEL SVM

### **Linear SVM:**

**Use When:**

- Data linearly separable
- High-dimensional sparse data (text)
- Very large datasets (fast training)
- Need interpretability
- Want feature coefficients

**Example:**


**Advantages:**

- Fast training O(n)
- Scales to large datasets
- Interpretable weights
- Lower memory usage

---

### **Kernel SVM (RBF, Polynomial):**

**Use When:**

- Data non-linearly separable
- Complex decision boundaries
- Smaller to medium datasets
- Don't need interpretability
- Have time for hyperparameter tuning

**Example:**

```python
from sklearn.svm import SVC

model = SVC(kernel='rbf', C=1.0, gamma='scale')
model.fit(X_train, y_train)
```

**Advantages:**

- Handles non-linear patterns
- Very flexible
- Often higher accuracy

**Disadvantages:**

- Slow training
- Memory intensive
- Black box

**Problem:** SVM training takes too long.

- Consider other algorithms (Random Forest, Logistic Regression)
- Increase cache_size parameter
- Use smaller portion of data for hyperparameter tuning

## TIME COMPLEXITY

### **Training:**

**Linear kernel:**

```
O(n × m)  (LinearSVC with liblinear)

Where:
- n = number of samples
- m = number of features

Fast! Suitable for large datasets
```

**Non-linear kernels (RBF, Polynomial):**

```
O(n² × m) to O(n³ × m)

Where:
- n² for kernel matrix computation
- n³ for quadratic programming
- m = number of features

Very slow for large n!
```

### **Prediction:**

```
O(n_sv × m)

Where:
- n_sv = number of support vectors
- m = number of features

Fast if few support vectors
```

### **Space Complexity:**

```
Training: O(n² + n × m)
- Kernel matrix: n²
- Feature matrix: n × m

Model storage: O(n_sv × m)
- Only need to store support vectors
```

**Example:**

```
10,000 samples:
- Kernel matrix: 10,000² = 100M entries
- At 8 bytes per float: ~800 MB
- Just for kernel matrix!

If 20% are support vectors:
- Model stores: 2,000 × m values
- Much smaller than full dataset
```

## BEST PRACTICES

1. **Always scale/normalize features**
    
    - Use StandardScaler
    - Fit on training data only
    - Critical for good performance
2. **Start with RBF kernel**
    
    - Best default choice
    - Try linear if RBF too slow
3. **Use grid search for hyperparameters**
    
    - C and gamma are critical
    - Use cross-validation
    - Start coarse, then refine
4. **Check for overfitting**
    
    - Compare train vs test accuracy
    - Count support vectors
    - Use cross-validation
5. **Handle imbalanced data**
    
    - Use class_weight='balanced'
    - Or SMOTE/undersampling
    - Use appropriate metrics (F1, not accuracy)
6. **For large datasets**
    
    - Use LinearSVC
    - Or SGDClassifier(loss='hinge')
    - Or sample your data
7. **Enable probability if needed**
    
    - Set probability=True
    - Adds training time
    - Needed for predict_proba()
8. **Monitor support vector count**
    
    - Should be 20-60% of training data
    - > 70%: Might be overfitting
        
    - <10%: Might be underfitting
9. **Use appropriate evaluation metrics**
    
    - Accuracy (balanced classes)
    - F1-Score (imbalanced classes)
    - ROC-AUC (probability estimates)
    - Confusion matrix
10. **Set random_state for reproducibility**
    
    - Ensures consistent results
    - Important for debugging

## COMMON PITFALLS

1. **Forgetting to scale features**
    
    - Most common mistake!
    - SVM will perform poorly
    - Always use StandardScaler
2. **Scaling before train-test split**
    
    - Causes data leakage
    - Scale after splitting
3. **Not tuning C and gamma**
    
    - Default values often suboptimal
    - Use grid search
4. **Using wrong kernel**
    
    - Linear for non-linear data
    - RBF for linearly separable data
    - Try both if unsure
5. **Ignoring training time**
    
    - RBF kernel very slow for large n
    - Use LinearSVC for n > 10,000
6. **Not handling imbalanced classes**
    
    - SVM biased to majority class
    - Use class_weight='balanced'
7. **Using accuracy on imbalanced data**
    
    - Misleading metric
    - Use F1-Score, ROC-AUC
8. **Overfitting with high gamma**
    
    - Decision boundary too complex
    - Decrease gamma or C
9. **Not checking convergence**
    
    - May not have fully trained
    - Increase max_iter if warnings
10. **Assuming SVM is best**
    
    - Not always optimal
    - Try Random Forest, XGBoost too
    - Especially for large datasets

## KERNEL APPROXIMATION FOR LARGE DATASETS

**Problem:** Kernel SVM too slow/memory-intensive for large datasets.

**Solution:** Approximate kernel function with explicit feature mapping.

### **RBFSampler (for RBF Kernel)**

```python
from sklearn.kernel_approximation import RBFSampler
from sklearn.linear_model import SGDClassifier

# Approximate RBF kernel with 100 dimensions
rbf_features = RBFSampler(gamma=0.1, n_components=100, random_state=42)
X_features_train = rbf_features.fit_transform(X_train)
X_features_test = rbf_features.transform(X_test)

# Train fast linear model
model = SGDClassifier(loss='hinge', max_iter=1000)
model.fit(X_features_train, y_train)
```

**Benefits:**

- Fast training (linear in n)
- Low memory
- Approximates RBF kernel

**Trade-off:**

- Slightly less accurate
- Need to choose n_components

---

### **Nystroem (General Kernel Approximation)**

```python
from sklearn.kernel_approximation import Nystroem
from sklearn.linear_model import SGDClassifier

# Approximate any kernel
nystroem = Nystroem(kernel='rbf', gamma=0.1, n_components=100)
X_features_train = nystroem.fit_transform(X_train)
X_features_test = nystroem.transform(X_test)

model = SGDClassifier(loss='hinge')
model.fit(X_features_train, y_train)
```

## COMPARING SVM IMPLEMENTATIONS

### **SVC (Support Vector Classification)**

- Full-featured SVM
- Supports all kernels
- Best for small-medium datasets (n < 10,000)
- Uses libsvm library

```python
from sklearn.svm import SVC
model = SVC(kernel='rbf', C=1.0, gamma='scale')
```

---

### **LinearSVC (Linear Support Vector Classification)**

- Optimized for linear kernel only
- Much faster than SVC(kernel='linear')
- Scales to large datasets
- Uses liblinear library
- No kernel trick overhead

```python
from sklearn.svm import LinearSVC
model = LinearSVC(C=1.0, max_iter=1000)
```

**When to use:** n > 10,000, linear decision boundary

---

### **NuSVC (Nu-Support Vector Classification)**

- Alternative parameterization
- Uses nu instead of C
- nu controls number of support vectors directly
- Rarely used in practice

```python
from sklearn.svm import NuSVC
model = NuSVC(nu=0.5, kernel='rbf')
```

---

### **SGDClassifier with hinge loss**

- Stochastic Gradient Descent approximation of linear SVM
- Very fast, scales to millions of samples
- Online learning (can train incrementally)
- Less accurate than LinearSVC

```python
from sklearn.linear_model import SGDClassifier
model = SGDClassifier(loss='hinge', max_iter=1000, tol=1e-3)
```

**When to use:** Very large datasets (n > 100,000)

---

|Implementation|Speed|Kernels|Scalability|Accuracy|Use Case|
|---|---|---|---|---|---|
|**SVC**|Slow|All|n < 10k|Best|Small datasets, non-linear|
|**LinearSVC**|Fast|Linear only|n < 100k|Good|Large datasets, linear|
|**SGDClassifier**|Very Fast|Linear only|Unlimited|Good|Huge datasets (millions)|
|**NuSVC**|Slow|All|n < 10k|Best|Research, specific nu requirement|
