## K-MEANS CLUSTERING - COMPREHENSIVE NOTES

**Definition:** An unsupervised learning algorithm that partitions a dataset into $k$ distinct, non-overlapping subgroups (clusters). It assigns each data point to the cluster whose center (centroid) is nearest to the point. The algorithm aims to minimize the variance within each cluster (intra-cluster variance).

**Used For:**

- Customer segmentation
    
- Image compression (color quantization)
    
- Pattern recognition
    
- Anomaly detection (points far from any centroid)
    
- Document clustering
    
- Social network analysis
    

**Cases of Application:**

- **Marketing:** Grouping customers by purchasing behavior for targeted ads.
    
- **Real Estate:** Clustering neighborhoods based on housing prices and crime rates.
    
- **IT:** Grouping computer network traffic to identify anomalies (DDoS attacks).
    
- **Insurance:** Identifying groups of motor insurance policyholders with a high average claim cost.
    
- **Astronomy:** Grouping stars or galaxies based on brightness and position.
    

**Prerequisites:**

- **Feature Scaling is mandatory:** Since K-Means uses distance (Euclidean) to assign points, features with larger ranges will dominate.
    
- **Specify K:** You must define the number of clusters ($k$) before training.
    
- **Handle Outliers:** K-Means is very sensitive to outliers (centroids get pulled away).
    
- **Symmetry:** Assumes clusters are spherical and roughly same size (struggles with irregular shapes).
    

**Performance Measures:**

- **Inertia (WCSS):** Sum of squared distances of samples to their closest cluster center. Lower is better (but 0 means overfitting).
    
- **Silhouette Score:** Measures how similar an object is to its own cluster compared to other clusters. Range: [-1, 1]. High value = well-separated clusters.
    
- **Davies-Bouldin Index:** Ratio of within-cluster distances to between-cluster distances. Lower is better.

## HOW K-MEANS WORKS (ALGORITHM INTUITION)

**The Iterative Process:**

1. **Choose K:** Select the number of clusters.
    
2. **Initialization:** Randomly select $k$ data points as initial centroids (centers).
    
3. **Assignment Step:** Calculate the distance (Euclidean) between each data point and all centroids. Assign the point to the closest centroid.
    
4. **Update Step:** Calculate the new centroid of each cluster by taking the **mean** of all data points assigned to that cluster.
    
5. **Convergence:** Repeat steps 3 and 4 until:
    
    - Centroids do not move.
        
    - Points do not change clusters.
        
    - Maximum iterations reached.
        

Mathematical Objective:

Minimize the Within-Cluster Sum of Squares (WCSS):

$$WCSS = \sum_{i=1}^{k} \sum_{x \in C_i} ||x - \mu_i||^2$$

- $C_i$: Cluster $i$
    
- $\mu_i$: Centroid of cluster $i$
    
- $x$: Data point

## CRITICAL SUBTOPICS

### **1. The Random Initialization Trap**

Problem:

The final clusters depend heavily on where the centroids are placed at the start. If you initialize centroids poorly (e.g., two centroids in the same natural cluster), the algorithm might get stuck in a "local optimum" and fail to find the true structure.

Solution: K-Means++

An intelligent initialization strategy (default in sklearn).

1. Choose the first centroid randomly from the data.
    
2. For each remaining data point, find the distance to the nearest existing centroid.
    
3. Choose the next centroid from the data points, where the probability of choosing a point is proportional to the square of the distance (points further away are more likely to be chosen).
    
4. Repeat until $k$ centroids are chosen.
    

_Result: Centroids are spread out initially, leading to faster convergence and better results._

---

### **2. Selecting the Number of Clusters (K)**

Since K is a hyperparameter, we use heuristics to find the "sweet spot."

#### **A. The Elbow Method**

- **Concept:** Run K-Means for a range of $k$ values (e.g., 1 to 10).
    
- **Plot:** K (x-axis) vs. WCSS (y-axis).
    
- **Interpretation:** As $k$ increases, WCSS decreases (more clusters = points closer to centroids). The "Elbow" is the point where the rate of decrease sharply slows down.
    
- **Formula:** Look for the "bend" in the knee. Adding more clusters beyond this point gives diminishing returns.
    

#### **B. Silhouette Analysis**

- **Concept:** Measures separation distance between closest clusters.
    
- **Formula:** $S = \frac{b - a}{\max(a, b)}$
    
    - $a$: Mean distance to other points in the _same_ cluster.
        
    - $b$: Mean distance to points in the _nearest neighboring_ cluster.
        
- **Interpretation:**
    
    - $+1$: Point is far away from neighboring clusters (Good).
        
    - $0$: Point is on or very close to the decision boundary.
        
    - $-1$: Point might have been assigned to the wrong cluster.

## IMPLEMENTATION CODE SNIPPETS

### **Standard K-Means with Elbow Method**

Python

```
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

wcss = []
# Try K from 1 to 10
for i in range(1, 11):
    kmeans = KMeans(n_clusters=i, init='k-means++', random_state=42)
    kmeans.fit(X_scaled)
    wcss.append(kmeans.inertia_)  # Inertia is the WCSS

# Plot the Elbow Graph
plt.plot(range(1, 11), wcss)
plt.title('The Elbow Method')
plt.xlabel('Number of clusters')
plt.ylabel('WCSS')
plt.show()
```

### **Training Final Model**

Python

```
# Assuming optimal K=3 from Elbow method
kmeans = KMeans(n_clusters=3, init='k-means++', random_state=42)
y_kmeans = kmeans.fit_predict(X_scaled)

# Getting Centroids
centroids = kmeans.cluster_centers_
```

## ADVANTAGES OF K-MEANS

1. **Simple & Understandable:** Easy to implement and interpret.
    
2. **Scalable:** Efficient for large datasets (Time complexity: $O(n \times k \times i)$ where $i$ is iterations).
    
3. **Fast:** Convergence is generally fast compared to other clustering algorithms.
    
4. **Adaptable:** Can warm-start centroids if new data comes in.
    

---

## DISADVANTAGES OF K-MEANS

1. **Manual Selection of K:** You must guess or test for $k$.
    
2. **Sensitive to Outliers:** One distant point can shift the mean significantly.
    
3. **Sensitive to Initialization:** Solved mostly by K-Means++.
    
4. **Spherical Assumption:** Assumes clusters are spherical (circle-like). Fails on elongated, crescent, or irregular shapes (use DBSCAN for those).
    
5. **Sensitive to Scale:** Must scale data (z-score usually).
    

---

## K-MEANS VS HIERARCHICAL CLUSTERING

|**Aspect**|**K-Means**|**Hierarchical Clustering**|
|---|---|---|
|**Type**|Partitional|Agglomerative/Divisive|
|**Data Size**|Good for Large Data|Hard for Large Data ($O(n^2)$ or $O(n^3)$)|
|**Shape**|Spherical only|Can handle arbitrary shapes (mostly)|
|**Hyperparameters**|Must define K|No K needed (cut dendrogram)|
|**Output**|Centroids & Labels|Dendrogram (Tree structure)|
|**Predictive?**|Yes (can predict new points)|No (usually descriptive only)|

---

## BEST PRACTICES

1. **Always Scale Data:** Use `StandardScaler`.
    
2. **Use K-Means++:** Never use random initialization unless for experimentation.
    
3. **Check Cluster Sizes:** If one cluster has 90% of data and others have 1%, K might be wrong or data needs cleaning.
    
4. **Visualize:** Use PCA or t-SNE to reduce data to 2D and visualize the clusters to verify separation.
    
5. **Restart:** Even with K-Means++, try `n_init=10` (default in sklearn) to run the algorithm 10 times and pick the best WCSS.
