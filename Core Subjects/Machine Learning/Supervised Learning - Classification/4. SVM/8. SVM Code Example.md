Here are the Python code snippets demonstrating the key processes in applying Support Vector Machines (SVM). I have broken this down by "process step" to clearly show how each decision affects the model's training and decision boundary.

We will use `scikit-learn` and a synthetic non-linear dataset (`make_moons`) to clearly visualize the effects.

### 1. The Critical Step: Feature Scaling

**Impact:** SVM calculates distances between data points. If one feature has large values (e.g., "Salary" = 50,000) and another small (e.g., "Age" = 25), the large feature will dominate the distance calculation, making the SVM biased and ineffective.

**Code:**

Python

```
from sklearn.datasets import make_moons
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC

# 1. Generate non-linear data
X, y = make_moons(n_samples=200, noise=0.2, random_state=42)

# 2. Split Data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 3. SCALING (CRITICAL FOR SVM)
# Without this, training takes longer and convergence is poor
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)
```

---

### 2. Kernel Selection (Handling Non-Linearity)

**Impact:** The kernel determines the "shape" of the decision boundary.

- **Linear:** Tries to draw a straight line (will fail on curved data).
    
- **RBF:** Can draw complex "islands" or curves around data.
    

**Code:**

Python

```
# A. Linear Kernel (Fast, but simple boundaries)
svm_linear = SVC(kernel='linear')
svm_linear.fit(X_train_scaled, y_train)

# B. RBF Kernel (Default, handles curves)
svm_rbf = SVC(kernel='rbf')
svm_rbf.fit(X_train_scaled, y_train)

# C. Polynomial Kernel (Curved, strictly defined by degree)
svm_poly = SVC(kernel='poly', degree=3)
svm_poly.fit(X_train_scaled, y_train)
```

---

### 3. Tuning Regularization (Parameter `C`)

**Impact:** Controls the "strictness" of the margin.

- **Low C:** "Soft Margin." Allows errors to keep the boundary simple. (Higher Bias).
    
- **High C:** "Hard Margin." Punishes errors heavily. Tries to classify every single point correctly. (Higher Variance/Overfitting).
    

**Code:**

Python

```
# Low C: Wider margin, accepts some misclassification (Generalizes better on noisy data)
svm_low_c = SVC(kernel='rbf', C=0.1)
svm_low_c.fit(X_train_scaled, y_train)

# High C: Narrow margin, strict classification (Risks overfitting outliers)
svm_high_c = SVC(kernel='rbf', C=1000)
svm_high_c.fit(X_train_scaled, y_train)
```

---

### 4. Tuning Sensitivity (Parameter `Gamma`)

**Impact:** Only used for RBF/Poly/Sigmoid. Controls how "far" the influence of a single training example reaches.

- **Low Gamma:** Far reach. The boundary is smooth and simple.
    
- **High Gamma:** Close reach. The boundary wraps tightly around individual data points (creating "islands").
    

**Code:**

Python

```
# Low Gamma: Smooth, gentle curve
svm_low_gamma = SVC(kernel='rbf', gamma=0.1)
svm_low_gamma.fit(X_train_scaled, y_train)

# High Gamma: Jagged, complex boundary (Overfitting risk)
svm_high_gamma = SVC(kernel='rbf', gamma=10.0)
svm_high_gamma.fit(X_train_scaled, y_train)
```

---

### 5. Visualization Helper (Seeing the Effect)

To actually _see_ these effects, you need to plot the decision boundary. This snippet creates a grid of points and predicts the class for every point to color the background.

**Code:**

Python

```
import numpy as np
import matplotlib.pyplot as plt

def plot_boundary(model, X, y, title):
    # Create a meshgrid (the background)
    x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.02),
                         np.arange(y_min, y_max, 0.02))

    # Predict across the entire grid
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)

    # Plot
    plt.contourf(xx, yy, Z, alpha=0.8, cmap=plt.cm.coolwarm)
    plt.scatter(X[:, 0], X[:, 1], c=y, edgecolors='k', cmap=plt.cm.coolwarm)
    plt.title(title)
    plt.show()

# Example usage:
plot_boundary(svm_high_gamma, X_train_scaled, y_train, "High Gamma (Overfitting)")
plot_boundary(svm_low_gamma, X_train_scaled, y_train, "Low Gamma (Underfitting/Smooth)")
```

### Summary of Effects

|**Process Step**|**Code Parameter**|**Effect on Training**|
|---|---|---|
|**Scaling**|`StandardScaler()`|Without this, gradient descent oscillates or fails; "large" features dominate the model.|
|**Kernel**|`kernel='rbf'`|Changes the _geometry_ of the boundary (Line vs. Curve).|
|**Regularization**|`C=100`|**Increase C** = More complex boundary, slower training, less bias, more variance.|
|**Sensitivity**|`gamma=10`|**Increase Gamma** = Boundary wraps tighter around points; model becomes very sensitive to noise.|
